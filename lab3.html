LAB 3 : Docker

Objectif

  • L’objectif principal de ce TP est de se familiariser avec la plateforme de conteneurisation d'applications Docker.

  • Pour ce faire, ce TP est organisé comme suit :

    • Mise en place et installation de Docker (sous Linux Ubuntu)
    • Gestion des images et des conteneurs
    • Manipulation d’un conteneur simple, Hello world
Prérequis
  • Pour installer Docker, vous devez posséder un système Linux 64 bits avec un noyau 3.10 au minimum.
    • Pour vérifier la version de votre noyau Linux, tapez sur le terminal la commande suivante :
In [ ]:
$uname -r
  • Les versions supportées pour Ubuntu sont : Bionic 18.04, Focal 20.04, Hirsute 21.04
  • Vous devez disposer des droits d'administration système.
  • Vous devez disposer d'une connexion à Internet configurée et activée.

Installation de Docker sur Ubuntu¶

  • Docker est un projet open-source sous licence Apache 2 pour automatiser le déploiement d'applications sous forme de conteneurs.

  • Il ya deux méthodes pour installer Docker sur Ubuntu :

    • Méthode 1 : installer Docker à partir du dépôt officiel Docker
    • Méthode 2 : installer Docker à partir du dépôt officiel de votre distribution Ubuntu

  • Pour installer Docker, vous devez choisir soit méthode 1, soit méthode2.

Méthode 1 : installation Docker depuis le dépôt officiel Docker¶

Étape 1 : installation des dépendances logicielles¶
  - Mettez à jour les informations sur les paquetages à partir des sources configurées.
In [ ]:
$sudo apt-get update

- Installez les outils suivants :
In [ ]:
$sudo apt-get install apt-transport-https
In [ ]:
$sudo apt-get install ca-certificates
In [ ]:
$sudo apt-get install curl
In [ ]:
$sudo apt-get install  gnupg-agent
In [ ]:
$sudo apt-get install software-properties-common
  • Explication
    • apt-transport-https : la méthode https est nécessaire pour permettre à APT d'utiliser un dépôt HTTPS (dépôt Docker).
    • ca-certificates (CA, Certification Authority): une autorité de certification (AC) est une entité chargée d'émettre des certificats numériques pour vérifier les identités des services tiers sur l'internet.
    • curl : utilitaire de ligne de commande permettant de transférer des données depuis ou vers un serveur conçu pour fonctionner sans interaction de l'utilisateur.
    • gnupg-agent (GNU Privacy Guard, GPG) : utilisé pour chiffrer les données échangées entre services.
    • software-properties-common : permet de gérer facilement les sources logicielles des fournisseurs de logiciels indépendants.
Étape 2 : Ajout clé Docker¶
  - Installez la clé GPG du dépôt Docker sur le serveur afin d’éviter le message d’avertissement indiquant que le dépôt n’est pas identifié.
In [ ]:
$curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
Étape 3 : Ajout du dépôt Docker aux sources¶
  - Ajoutez le dépôt de Docker à la liste des référentiels dans le fichier "sources.list".
In [ ]:

   "deb [arch=amd64] https://download.docker.com/linux/ubuntu \
   $(lsb_release -cs) \
   stable"

Accord des privilèges nécessaires¶

  • Pour qu'un utilisateur puisse se servir de la commande "docker", il doit appartenir au groupe "docker".

  • Le groupe "docker" accorde des privilèges équivalents à l'utilisateur root

  • Ainsi, pour éviter d’avoir à utiliser « sudo » lorsque vous utilisez le docker, vous devez appartenir au groupe "docker"

      - Si ce groupe n'existe pas encore sur votre ordinateur, vous devez le créer avec la commande "groupeadd".
In [ ]:
$sudo groupadd docker

-  Ajoutez ensuite l'utilisateur au groupe, avec la commande "usermod".
In [ ]:
$sudo usermod -aG docker $USER
  • L'option "-aG" permet d'ajouter un utilisateur au groupe qui suit. La variable d'environnement "$USER" contient l'utilisateur en cours.
  • Redémarrez pour que les modifications

Gestion des images¶

1- Lister des images¶
  • Pour voir la liste des images disponibles en local, tapez la commande :
In [ ]:
$docker images

- Résultat commande
In [ ]:
$ docker images
REPOSITORY    TAG       IMAGE ID       CREATED       SIZE
hello-world   latest    feb5d9fea6a5   2 weeks ago   13.3kB
  • REPOSITORY : le nom de l’image dans le dépôt.
  • TAG : sa version.
  • IMAGE ID : un identifiant unique pour l’image
  • CREATED : la date de création de l’image dans la machine hôte de docker. Attention, il ne s’agit pas de la création de l’image par le dépôt, mais bien de celle de la création du cache localement.
  • SIZE : la taille de l’image.

In [ ]:
$docker search <image_name>

4- Supprimer une image¶
  • Pour supprimer une image du dépôt local, sans conteneur démarré, on utilise la commande "rmi".
In [ ]:
$docker rmi <image ID_or_Name>
Exemple¶
  • Supprimez l'image centos.
In [ ]:
$docker rmi feb5d9fea6a5

ou

In [ ]:
$docker rmi centos
  • Pour supprimer toutes les images, on tape:
In [ ]:
$docker rmi $(docker images -a -q)

CONTAINER ID   IMAGE           COMMAND       CREATED          STATUS        PORTS     NAMES
c8863573b026   centos:latest   "/bin/bash"   6 seconds ago    Created                 MyFirstContainer

  • Reprenons les différentes informations :

    • CONTAINER ID: l’identifiant unique du conteneur. Il est créé à chaque fois du lancement du conteneur, même pour instancier la même image
    • IMAGE : il s’agit du nom de l’image utilisée pour instancier le conteneur
    • Le suffixe : latest indique que c’est la dernière version disponible (par défaut)
    • COMMAND: cette information reprend le contenu de la ligne CMD du fichier Dockerfile
    • CREATED : donne la date d’initialisation du conteneur
    • STATUS : contient plusieurs informations. Dans notre cas, le conteneur s’est arrêté, et le processus interne est sorti avec le code 0, qui correspond par convention à un déroulement sans erreur.
    • PORTS : les ports réseaux exposés par le conteneur, en l’occurrence aucun dans le cas de ce simple exemple n’affichant que du texte
    • NAMES : un nom est affecté au conteneur lors de son lancement, pour pouvoir le manipuler plus facilement que par son identifiant

7- Supprimer un conteneur¶
  • La commande "docker rm" permet de supprimer un ou plusieurs conteneurs spécifiques (par nom ou identifiant).
In [ ]:
$docker rm <Container ID_or_Name>
  • Pour supprimer plusieurs conteneurs
In [ ]:
$docker rm <container ID_or_Name>  <container ID_or_Name>
  • Par défaut, docker ne supprime pas un conteneur en cours d'exécution. Pour supprimer un conteneur, quel que soit son état, on tape :
In [ ]:
$docker rm -f <container ID_or_Name>
  • Pour supprimer un conteneur automatiquement à sa fermeture (après son exécution) :
In [ ]:
$docker run --rm <image_name>
  • Pour supprimer tous les conteneurs, vous pouvez utiliser cette instruction :
In [ ]:
$ docker rm $(docker ps -a -q)
  • Pour supprimer tous les conteneurs, quel que soient leurs états :
In [ ]:
$ docker rm -f $(docker ps -a -q)

Image Docker, hello-world¶

  • Comme toute technologie informatique, DOCKER dispose d’un exemple "Hello World", soit dans notre cas un conteneur qui ne fait rien à part afficher un message de bienvenue.
  • L’intérêt de ce conteneur est de vérifier que la plateforme fonctionne correctement.
  • Pour utiliser l'image "hello-word", on va l'exécuter sur un conteneur en utilisant la commande suivante :
In [ ]:
$docker run hello-world
  • La commande "run" charge une image et l'exécute sur un nouveau conteneur.

Fichier Dockerfile "hello-world"¶

  • Faites l'accès aux branches "https://github.com/docker-library/hello-world/tree/master/amd64/hello-world" et "https://github.com/docker-library/hello-world/tree/master/i386/hello-world" et contenu du répertoire.

  • Le répertoire courant contient le fichier image Dockerfile et le fichier code objet "hello".
  • Contenu du fichier image Dockerfile.
In [ ]:
# Contenu du fichier Dockerfile
FROM scratch
COPY hello /
CMD ["/hello"]

  • Explication

    • La première ligne définit sur quelle image de base / plateforme la présente image va se baser. Dans notre cas, le mot-clé scratch correspond à une machine vide.
    • La seconde ligne fait en sorte que, lorsque le fichier Dockerfile sera lu pour générer une image, le fichier objet "hello" sera ajouté dans la racine du système de fichiers de l’image.
    • La troisième établit que le démarrage de l’image provoquera le lancement de la commande /hello, qui exécutera le contenu du fichier objet précédemment recopié (hello).
