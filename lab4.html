Objectif

Construction d'une image Docker "Hello"

Dveloppement d'une application "hello" sur la machine locale

1- Crez un rpertoire "hello_directory" qui va contenir votre dockerfile.

In [ ]:
$ mkdir hello_directory
$ cd hello_directory

2- Dans le rpertoire hello, crez un fichier "hello.c" qui permet dafficher le message "Hello From Docker LAB !".

In [ ]:
$ nano hello.c
In [ ]:
#include <stdio.h>
int main()
{
   printf("Hello From Docker LAB !  \n");
    return 0;
    
}

3- Compilez le code source (hello.c) avec gcc en crant le fichier excutable "hello"

In [ ]:
$ gcc -o hello -static hello.c

- L'option "static" est ajoute pour crer un fichier excutable statique sans dpendances logicielles. En d'autres termes, il ne ncessite pas de dpendance sur les bibliothques dynamiques au moment de l'excution de la commande.

4- Affichez le contenu du dossier hello aprs compilation : un fichier excutable "hello" s'est ajout aprs compilation (fichier objet).

In [ ]:
$ ls
hello  hello.c

5- Excutez le fichier "hello"

In [ ]:
$ ./hello
In [ ]:
## Rsultat
Hello From Docker LAB !

Cration d'une image Docker "hello" partir du fichier Dockerfile : command "docker build"

  • On va crer une image Docker "hello" qui renvoie un message d'accueil sur le terminal "Hello From Docker LAB !"
  • Pour crer cette image Docker, on va prparer le fichier Dockerfile de l'image.
  • Le fichier Dockerfile doit aboutir une image excutable de l'application. Il doit comporter une description de toutes les dpendances de l'image ainsi que les commandes ncessaires pour l'excuter.
  • Pour ce faire, on doit spcifier dans le fichier Dockerfile, notre plateforme de dpart. En d'autre termes, on doit spcifier le systme de fichiers sur le quel on va former notre application.
  • Pour notre application "hello" deux choix sont possibles pour excuter l'application et afficher le message d'accueil :
    • partir du fichier code source "hello.c" : image de base ubuntu
    • partir du fichier code objet "hello.o" : image de base machine vide : scratch puisque ce fichier est indpendant de toute plateforme logicielle et pour excuter la commande ./hello, un noyau n'a besoin d'aucune dpendance logicielle : aucune image de base n'est requise.

Mthode 1 : image de base "ubuntu"

  • Pour diter le fichier Dockerfile en partant d'un systme de fichiers "ubuntu", on doit enchainer avec ces actions :
    • Installer le compilateur gcc
    • Copier le code source "hello.c" sur le systme de fichiers image
    • Compiler le fichier code source
    • Excuter la commande "/hello"
Etape 1 : Prparation du fichier Dockerfile

1- Tapez la commande suivante pour crer et diter le fichier "Dockerfile" :

In [ ]:
$ nano Dockerfile

2- Ecrivez les lignes de code suivante sur le fichier Dockerfile.

In [ ]:
FROM ubuntu
RUN apt-get -y update
RUN apt-get -y install gcc
COPY hello.c /
RUN gcc -o hello hello.c
CMD ["/hello"]
Etape 2 : Cration image avec la commande "docker build"

3- Crez une image nomme hello:version1 partir du Dockerfile. - Noubliez pas d'ajouter le point en fin de ligne pour indiquer le chemin d'accs du fichier Dockerfile.

In [ ]:
$ docker build -t hello:version1 .

4- Vrifiez que l'image est cre

In [ ]:
$ docker images
In [ ]:
## Rsultat
~/hello_directory$ docker images
REPOSITORY    TAG        IMAGE ID       CREATED         SIZE
hello   version1   bbd500eb2345   5 seconds ago   251MB
ubuntu        latest     ba6acccedd29   12 days ago     72.8MB

Etape 3 : Excution d'un conteneur partir de l'image "hello"

5- Crer et dmarrer un conteneur qui utilise l'image "hello"

In [ ]:
$ docker run hello:version1
In [ ]:
#Rsultat

Hello From Docker LAB !

Etape 1 : Cration et dmarrage d'un conteneur Docker

1- Crez un nouveau conteneur Docker partir de l'image "ubuntu" en mode intractif et nommez le "hello_container"

In [ ]:

3- Faites l'accs votre conteneur en ouvrant une session bash :

5- Installez gcc :
